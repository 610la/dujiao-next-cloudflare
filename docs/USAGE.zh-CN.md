# 使用说明

## 后台登录

后台地址由 `config/project.json` 的 `adminPath` 决定。项目没有默认账号，必须先按部署文档运行管理员创建脚本。

登录后建议立即启用管理员两步验证，并把恢复码离线保存。

## 站点设置

在系统设置中配置：

- 站点名称、Logo、图标和正式站点 URL
- SEO 标题、关键词和描述
- 默认语言、币种和时区
- 首页公告、页面内容和模板
- 邮件、验证码和订单通知策略
- Cloudflare Turnstile

站点 URL 会用于邮件链接、回调地址和分站识别，必须填写实际线上地址。

## 商品与库存

推荐顺序：

1. 创建分类。
2. 创建商品并填写多语言名称、描述和详情。
3. 创建一个或多个 SKU，设置价格与成本价。
4. 选择人工交付或自动交付。
5. 设置库存展示方式、购买上下限和批发价阶梯。
6. 上架前使用测试账号完成一次下单。

人工交付商品使用 SKU 的人工库存。自动交付商品需要先导入卡密；可用卡密数量就是可售库存。订单创建、锁单、支付和交付会分别更新锁定、已售和可用数量。

## 人工交付

商品可配置收集字段，例如联系方式、账号信息、收件人和地址。用户提交的内容会随订单快照保存。

支付成功后，人工订单进入处理中。管理员填写发货内容或物流信息并交付；完成后再把订单标记为已完成。不要在发货内容中粘贴不需要暴露给用户的内部备注。

## 自动交付

先创建卡密批次并绑定商品与 SKU，再导入卡密。系统只会分配状态为可用且未锁定的卡密。

上线前验证：

- 单件与多件购买
- 最后一件库存的并发下单
- 支付超时释放库存
- 重复支付回调的幂等处理
- 用户订单页面的复制内容

## 支付渠道

在支付渠道中选择对应提供商并填写商户参数。密钥只保存在后台，不要写入源码。

每个渠道至少测试：

- 发起支付
- 浏览器同步返回
- 服务端异步通知
- 重复通知
- 金额校验
- 订单超时
- 退款流程

只有收到签名正确、商户订单号正确、金额和币种一致的服务端通知后，订单才应被视为支付成功。

## 邮件

支持后台配置 Resend 或 Cloudflare Email。先验证发信域名，再填写发件地址和 API Key，并使用测试发送功能确认。

生产环境建议只在重要状态变化时发送订单邮件，例如已交付、已完成、已退款；验证码邮件需要配置发送间隔和尝试次数限制。

## 订单状态

- 待支付：订单已创建但尚未确认付款。
- 已支付：付款已确认，尚未开始人工处理。
- 处理中：人工交付或采购流程正在进行。
- 部分交付：订单只完成了部分子订单。
- 已交付：交付内容或物流信息已经提交。
- 已完成：订单流程结束。
- 已取消：订单取消并释放仍锁定的资源。
- 已退款 / 部分退款：退款流程已经执行。

不要直接通过数据库修改订单状态，应使用后台操作，让库存、钱包、卡密和通知副作用保持一致。

## 用户与游客订单

注册用户只能查看自己的订单。游客需要保存订单号、查询邮箱和订单查询密码。不要把游客查询凭据放在公开页面或客服群中。

## 运维建议

- 定期导出 D1 数据库并测试恢复。
- 定期检查 Worker 错误日志、支付失败和邮件失败记录。
- 更新前先备份，更新后执行迁移、测试和小额支付验证。
- 定期更换管理员密码并审计管理员角色。
